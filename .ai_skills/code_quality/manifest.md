# 🧹 Code Quality — "Чистота Кода"

**Навык:** Автоматическая проверка стиля, форматирования и качества кода

---

## 🎯 Назначение

Этот навык обеспечивает:
- **Единый стиль** кода во всём проекте
- **Автоматическое форматирование** по правилам
- **Поиск багов** до запуска кода
- **Читаемость** для всей команды
- **Предотвращение** ошибок в продакшене

---

## 🎯 Триггеры для Использования

**Используй этот навык, когда:**

1. **Перед коммитом**
   - "Проверь код перед коммитом"
   - "Отформатируй файлы"
   - "Найди проблемы в коде"

2. **Новый разработчик**
   - "Настрой линтер для проекта"
   - "Добавь правила форматирования"
   - "Создай конфиг для команды"

3. **Рефакторинг**
   - "Найди неиспользуемые переменные"
   - "Проверь сложность функций"
   - "Укажи на проблемы с производительностью"

4. **Code Review**
   - "Проверь пул-реквест"
   - "Найди нарушения стиля"
   - "Оцени качество кода"

---

## 📦 Зависимости

**Основные:**
- **ESLint** — линтер для JavaScript
- **Prettier** — автоматическое форматирование

**Дополнительно:**
- **eslint-config-prettier** — отключение конфликтующих правил
- **eslint-plugin-import** — проверка импортов

**Установка:**
```bash
npm install -D eslint prettier eslint-config-prettier
```

---

## 🚀 Быстрый Старт

**1. Инициализация:**
```bash
npx eslint --init
```

**2. Запуск проверки:**
```bash
npx eslint .
npx eslint . --fix  # Автоматическое исправление
```

**3. Форматирование:**
```bash
npx prettier --write "**/*.js"
```

---

## 📊 Примеры Использования

### 1. Проверка файла
```bash
npx eslint server.js
```

**Вывод:**
```
✖ 3 problems (1 error, 2 warnings)
```

### 2. Авто-исправление
```bash
npx eslint . --fix
```

**Результат:**
- ✅ Исправлены отсутствующие точки с запятой
- ✅ Добавлены недостающие кавычки
- ✅ Удалены неиспользуемые переменные

### 3. Форматирование проекта
```bash
npx prettier --write "**/*.{js,jsx,ts,tsx}"
```

**Что делает:**
- Выравнивает отступы
- Добавляет/удаляет пробелы
- Переносит длинные строки

## 🎯 Правила Проекта

**Наши стандарты:**
- **Отступы:** 2 пробела
- **Кавычки:** одинарные (`'`)
- **Точки с запятой:** обязательно
- **Длина строки:** 100 символов
- **Конечная запятая:** в многоуровневых объектах
